🪪**Task Tracker API**🪪 

Приветствую вас! Перед вами REST API - Task Tracker API - сервис, предназначенный для создания задач, у которых имеется уникальный идентификатор, наименование,
описание, статус (имеется 4 состояния статуса: "new", "cancelled", "in_progress", "done"), дата создания и дата обновления. Вы можете создавать новую задачу,
менять ее, менять ее параметры. Сервис реализован на языке программирования Go с использованием Echo, Docker, PostgreSQL, GORM (для ускорения разработки
и упрощения самого написания).

**Что сделать для запуска?**
- 1. Установите Docker Desktop
  2. Выполните команду в терминале: git clone <*ссылка на мой проект*>
  3. Перейдите в скачанную директорию и выполните команду: docker compose up --build
  4. После некоторого промежутка времени сервис запустится на вашем компьютере
  5. Выполните тестирование с помощью программы Postman (можете также использовать консольный curl, зависит от ваших личных предпочтений)
  6.  Введите в Postman: http://localhost:8080/api/health для проверки работоспособности сервиса (можете ввести http://localhost:8080/api/ping, получите
  название известной игры
  8. Введите в Postman: http://localhost:8080/api/tasks, выбрав GET. Вы получите список всех задач, очевидно, в начале этот список будет пуст.
  9. Возможна выборка записей по определённому идентификатору: http://localhost:8080/api/tasks/:id
  10. Протестируйте другие методы

  Пожалуй, на этом всё :)

  *И, конечно же, ошибок в коде сервиса не избежать ;) если таковые найдутся, то сообщите*
